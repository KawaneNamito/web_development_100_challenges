## 依頼事項

<details>

<summary>【依頼#01】</summary>

> YouTube での配信を楽にするツールを作りたいんですよね。
>
> フロントエンドはそのうち私が作るので、皆さんにはバックエンドの API を作って欲しいです。
>
> で、とりあえずの困りごとなんですけど、配信の概要欄を再利用できるようにしたいです。
>
> YouTube の配信って過去の履歴を再利用して立てることができるんですけど、ストックは何十個かで上限があるんですよね。雑談配信とか繰り返していると数カ月しか遡れないんですよね。
>
> 概要欄って挨拶や配信の注意事項の他にゲームによっては細かい規約を書いたり、雑談配信でも BGM や画像素材のクレジットを入れたりとか一回書くのに 5 分から 15 分位かかるんですよ。
>
> それに数年前に配布された素材とかだともうクレジットを見つけられなかったりすることもあります。
>
> なので概要欄の情報をなるべく長く保持しておいて、あとから取得できるようなサービスを作りたいんです。
>
> 欲しい情報としてはこんな感じですかね。
>
>     配信タイトル
>     データ登録時刻
>     概要欄の内容
>     1 ~ 5000文字
>
> とりあえず今はローカルホストで検証用の API が立てられればよいかな、と思っています。
>
> API のレスポンスは JSON が楽かな。
>
> 機能としては
>
>     登録
>     タイトル・時刻一覧
>     詳細（概要欄の内容）取得
>     削除
>
> ができると嬉しいです。
>
> そのうち Google ログインか Twitter ログインを実装したい気がするけど、最初から認証必須にするとデバッグしづらいしログインは一旦なしでもいいかなぁ。ユーザー ID はいい感じにリクエストに含めることにして……。
>
> うん、とりあえずそんなの作ってもらったらフロントエンド実装しますね！
>
> 使っているうちに新しい要望とか出すかもしれないです！

</details>

<details>

<summary>【依頼#02】</summary>

> 【依頼#02】
> 回答したい人用フォーム: https://docs.google.com/forms/d/e/1FA...
>
> とりあえずどういう系の配信なのかわかればマシになると思うので、
> 登録内容に「カテゴリ」みたいなのを追加できないですかね？
>
> カテゴリの例
>
> 雑談
> ゲーム（スプラ）
> ゲーム（APEX）
> …
>
> みたいな感じで。
>
> で、取得のときにはそのカテゴリも返してほしいです。
>
> あと、一覧取得のときにカテゴリを指定して絞り込む機能も欲しいです！
> あとあと、指定した件数分だけ取れるようにしてもらえると嬉しいです。
>
> 指定件数に収まらなかった場合は後続のデータを取得する方法も用意してもらえると助かります！
>
> 雑談カテゴリで 50 件要求して、51 件以上データが存在したらまずは先頭の 50 件、で、その次は 51 件目から……みたいな感じですね！
>
> 次のデータをどうやって取ったらいいかとか、データを最後まで取りきれたかとかが分かりやすい感じのレスポンスになっていると嬉しいです！
>
> あ、それと削除機能について説明が足りてなかったかもしれないんですけど念の為論理削除でお願いしたいです！まぁこれはやってありますかねｗ
>
> ちょっと API の使い方が難しくなりそうなので、何か API の仕様がわかるドキュメントもおいてもらえると嬉しいです。

```bash

cd backend-golang && go run ./cmd/api  # API サーバー起動 (localhost:8080)
```

Swagger UI: http://localhost:8081
