<summary>【依頼#01】</summary>

> YouTube での配信を楽にするツールを作りたいんですよね。
>
> フロントエンドはそのうち私が作るので、皆さんにはバックエンドの API を作って欲しいです。
>
> で、とりあえずの困りごとなんですけど、配信の概要欄を再利用できるようにしたいです。
>
> YouTube の配信って過去の履歴を再利用して立てることができるんですけど、ストックは何十個かで上限があるんですよね。雑談配信とか繰り返していると数カ月しか遡れないんですよね。
>
> 概要欄って挨拶や配信の注意事項の他にゲームによっては細かい規約を書いたり、雑談配信でも BGM や画像素材のクレジットを入れたりとか一回書くのに 5 分から 15 分位かかるんですよ。
>
> それに数年前に配布された素材とかだともうクレジットを見つけられなかったりすることもあります。
>
> なので概要欄の情報をなるべく長く保持しておいて、あとから取得できるようなサービスを作りたいんです。
>
> 欲しい情報としてはこんな感じですかね。
>
>     配信タイトル
>     データ登録時刻
>     概要欄の内容
>     1 ~ 5000文字
>
> とりあえず今はローカルホストで検証用の API が立てられればよいかな、と思っています。
>
> API のレスポンスは JSON が楽かな。
>
> 機能としては
>
>     登録
>     タイトル・時刻一覧
>     詳細（概要欄の内容）取得
>     削除
>
> ができると嬉しいです。
>
> そのうち Google ログインか Twitter ログインを実装したい気がするけど、最初から認証必須にするとデバッグしづらいしログインは一旦なしでもいいかなぁ。ユーザー ID はいい感じにリクエストに含めることにして……。
>
> うん、とりあえずそんなの作ってもらったらフロントエンド実装しますね！
>
> 使っているうちに新しい要望とか出すかもしれないです！
